NOVA CONVENÇÃO DE NOMENCLATURA (v0.3+):
═══════════════════════════════════════════════════════════════
Para evitar conflitos entre packages, todos os elementos transformados 
para JavaScript devem incluir o nome do package além do prefixo:

Formato: <PREFIXO>_<PACKAGE>_<ELEMENTO>

Exemplos:
- Package: SmartHome, Component: AirConditioner → CP_SmartHome_AirConditioner
- Package: Industrial, Enum: States → EN_Industrial_States  
- Package: Sensors, ValueType: Temperature → VT_Sensors_Temperature
- Package: Control, Activity: Monitor → AC_Control_Monitor

Benefícios:
✅ Elimina conflitos entre packages diferentes
✅ Facilita identificação da origem do elemento
✅ Mantém namespace claro e organizado
✅ Permite reutilização segura de nomes
